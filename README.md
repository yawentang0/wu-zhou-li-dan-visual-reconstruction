# wu-zhou-li-dan-visual-reconstruction

《武周·皇嗣李旦》历史考据向影视化重构项目

以武周时期的李旦为核心，围绕武则天、李显、太平公主、李隆基、安金藏等历史人物，建立一套公开、可追溯、可继续扩展的历史考据与影视化开发文档。

## 项目为什么存在

武周时期常被简化为“女皇传奇”或“宫廷权谋”，但这一时期真正复杂之处，在于唐宗室、武氏集团、皇权合法性、礼制重塑和亲族关系同时被压进同一个政治空间。

李旦尤其适合作为切入口。他不是单纯的旁观者，也不是只有文艺气质的退让皇子。他曾为帝、被降为皇嗣、长期处在权力中心与危险边缘之间，其沉默、谨慎、退让和再次进入政治舞台的过程，包含着极强的历史张力。

本项目希望在史料、制度、视觉和叙事之间建立桥梁：先把历史环境和人物处境研究清楚，再将其转化为人物设定、场景图、分镜、剧本和后续视频化流程。

## 项目目录说明

- `00_project_management/`：任务板、版本记录、决策记录。
- `01_research/`：资料来源、年表、人物索引、政治背景、礼仪制度、服饰、建筑与城市研究。
- `02_characters/`：武则天、李旦、李显、李隆基、太平公主、安金藏等人物设定与关系图。
- `03_worldbuilding/`：明堂、洛阳宫、长安与洛阳街市、宫廷礼仪和视觉规则。
- `04_story/`：故事圣经、季纲、单集大纲、场景表与剧本草稿。
- `05_visual_development/`：角色设计、服饰设计、建筑设计、场景概念、分镜和图像提示词。
- `06_video_pipeline/`：动画计划、镜头表、对白与配音、音乐声音和工具测试记录。
- `assets/`：参考资料、生成图像、图表和导出文件。

## 创作原则

- 以史料和研究为基础，不用现代爽剧逻辑替代历史处境。
- 不把历史人物脸谱化，不把武则天简化为暴君，也不把李旦简化为无害文人。
- 审美上追求克制、压迫、礼制感和政治重量，避免仙侠化、游戏化和过度奇观化。
- 视觉设计必须服务于制度、身份和场景功能，而不只是追求华丽。
- 所有重要创作处理都应标注依据或说明其虚构性质。

## 史实、推测与戏剧创作的区分规则

本项目文档会尽量使用以下标记：

- `史料明确记载`：可在正史、编年、诏令、碑铭、墓志或其他可靠材料中找到明确依据。
- `考古或图像资料支持`：由遗址、器物、壁画、墓葬、图像或博物馆藏品提供支持。
- `合理推测重构`：史料未直接说明，但可根据制度、时代环境、人物处境和同类材料推导。
- `纯戏剧创作`：为影视叙事、人物冲突或场景调度而设置，不能当作史实使用。

当同一内容包含多种层级时，应优先说明最可靠的依据，再说明推测或戏剧处理的范围。

## 第一阶段目标

第一阶段重点不是完成剧本，而是建立可靠的开发底座：

- 完成武则天、李旦、李显、太平公主、李隆基、安金藏的人物设定初稿。
- 整理李旦从唐宗室、皇帝、皇嗣到再度进入权力结构的关键年表。
- 梳理武周时期宫廷礼制、称谓、服饰、建筑空间和政治关系。
- 建立第一版视觉规则，明确哪些元素可用、慎用或禁用。
- 形成第一场戏《明堂觐见》的场景研究、人物关系和视觉方向。

## 后续计划

项目后续将逐步推进到：

- 人物关系图和政治势力图。
- 神都洛阳、明堂、洛阳宫和长安空间研究。
- 角色造型、服饰、建筑、场景概念图方向。
- 第一季故事圣经、分集大纲和第一集剧本草稿。
- 分镜、镜头表、声音设计与视频化测试流程。
